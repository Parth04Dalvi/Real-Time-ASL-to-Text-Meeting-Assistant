ASL Real-Time Meeting Assistant 🤟<p align="center"><b>A browser-native, edge-AI solution for real-time American Sign Language (ASL) translation during video conferences.</b></p> Overview<p>This project is a <b>Chrome Extension</b> designed to improve accessibility in digital workspaces. By leveraging <b>TensorFlow.js</b> and <b>Mediapipe</b>, it translates ASL gestures into text captions directly within the browser, providing a seamless overlay for platforms like Google Meet and Zoom.</p>Technical Implementation<p>

Building on my research in <b>Audio-Based Object Detection</b> and <b>YOLOv8</b>, this project focuses on optimizing <b>Machine Learning</b> models for client-side execution to ensure user privacy and zero server latency.
</p><b>Edge AI Inference:</b> Utilizes a custom-trained lightweight Neural Network exported via <b>tfjs-converter</b> to run inference on the user's local GPU.<b>Hand Tracking:</b> Implements <b>Mediapipe's</b> hand landmark detection to extract 63 spatial coordinates (21 points in 3D space) per frame.<b>Real-Time UI:</b> Developed with <b>React</b> and <b>Tailwind CSS</b>, utilizing a <b>Shadow DOM</b> to inject captions into third-party video call interfaces without style conflicts.<b>Performance Optimization:</b> Achieved sub-30ms inference latency, reflecting my experience in reducing <b>database latency</b> by 20% in professional environments. Features<b>Privacy-First:</b> No video data is sent to a server; all processing happens locally on the device.<b>Dynamic Vocabulary:</b> Supports the ASL alphabet (A-Z) and functional gestures like <b>Space</b> and <b>Backspace</b> for sentence construction.<b>Cross-Platform:</b> Compatible with any Chromium-based browser (Chrome, Edge, Brave).
